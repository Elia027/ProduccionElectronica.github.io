KiCad

Software de diseño de esquemas electrónicos y placas de circuito impreso (PCB) de código abierto y multiplataforma.

Volver al Inicio

¿Qué es KiCad?

KiCad es un software de diseño electrónico (EDA: Electronic Design Automation) de código abierto y multiplataforma, disponible para Windows, Linux y MacOS. Su objetivo principal es permitir a ingenieros, estudiantes y aficionados crear desde esquemáticos electrónicos hasta placas de circuito impreso (PCB) listas para fabricación.

Herramientas:

Editor de Esquemas

para capturar circuitos electrónicos, organizar jerárquicamente el diseño y comprobar su lógica eléctrica.

Editor de PCB (Pcbnew)

Convierte esquemáticos en placas impresas, diseñar pistas, colocar componentes y optimizar el enrutamiento.

Visualizador 3D

Verifica cómo se verá la placa y al mismo tiempo detectar posibles interferencias mecánicas.

Gerber Generation y Drill

Son formatos estándar que permiten mandar el diseño a fabricación.

Simulación SPICE y bibliotecas de componentes

Incluyen símbolos electrónicos, huellas físicas (footprints) y modelos 3D.

Código Abierto

Software libre y gratuito con una comunidad activa de desarrollo.

Flujo de Trabajo en KiCad

El proceso de diseño con KiCad generalmente sigue estos pasos:

  • Creación del esquema electrónico con el editor de esquemas (Eeschema)
  • Asignación de footprints a los componentes
  • Generación de la netlist
  • Diseño del PCB con el editor de circuitos impresos (Pcbnew)
  • Colocación de componentes y enrutamiento de pistas
  • Verificación de reglas de diseño (DRC)
  • Generación de archivos de fabricación (Gerber, drill, etc.)
  • Visualización 3D del diseño final

Editor de Esquemáticos (Eeschema)

El editor de esquemáticos es donde comienza todo diseño en KiCad. Permite crear representaciones gráficas de circuitos electrónicos utilizando símbolos de componentes interconectados.

Funciones principales:

  • Colocación de componentes: Añadir símbolos electrónicos desde bibliotecas
  • Conexión de componentes: Utilizando cables (wires) o etiquetas (labels)
  • Jerarquía: Crear esquemáticos multinivel con hojas anidadas
  • Anotación: Asignar referencias únicas a cada componente (R1, R2, C1, etc.)
  • Verificación eléctrica (ERC): Detectar errores en las conexiones
  • Generación de netlist: Crear lista de conexiones para el PCB

Barra de herramientas y menús esenciales:

Place → Wire Conectar componentes con cables
Place → Label Añadir etiquetas para conexiones con nombre
Place → Power Port Añadir símbolos de alimentación (VCC, GND)
Place → No Connect Flag Marcar pines intencionalmente desconectados
Tools → Annotate Schematic Asignar referencias a componentes
Tools → Electrical Rules Checker Verificar errores eléctricos en el esquema

Editor de PCB (Pcbnew)

Pcbnew es el editor de placas de circuito impreso donde se diseña el layout físico basado en el esquemático.

Funciones principales:

  • Colocación de componentes: Organizar componentes en la placa
  • Enrutamiento: Conectar componentes con pistas de cobre
  • Diseño de zonas: Crear áreas de cobre (pour) para alimentación o tierra
  • Verificación de reglas (DRC): Asegurar que el diseño cumple con las reglas establecidas
  • Generación de archivos de fabricación: Crear Gerbers, archivos de taladro, etc.
  • Visualización 3D: Ver el diseño en tres dimensiones

Barra de herramientas y menús esenciales:

Menú/Herramienta Función
File → Load Netlist Cargar conexiones desde el esquemático
Place → Footprint Añadir componentes manualmente
Route → Interactive Routing Enrutar pistas entre componentes
Route → Differential Pair Enrutar pares diferenciales
Place → Zone Crear zonas de cobre (pour)
Place → Via Insertar vías para cambiar de capa
Place → Keepout Area Definir áreas restrictivas
Inspect → Measure Medir distancias entre puntos
Tools → Design Rules Checker Verificar que el diseño cumple las reglas
Tools → Update PCB from Schematic Sincronizar cambios del esquemático
View → 3D Viewer Visualizar el diseño en 3D
File → Fabrication Outputs Generar archivos para fabricación (Gerbers, drill, etc.)

Capas importantes en PCB design:

  • F.Cu/B.Cu: Capas de cobre frontal y posterior
  • F.Silkscreen/B.Silkscreen: Serigrafía para marcar componentes
  • F.Mask/B.Mask: Máscara de soldadura
  • Edge.Cuts: Línea de corte que define el contorno de la placa
  • Dwgs.User: Capa para notas y dibujos

Reglas de diseño

Para garantizar la funcionalidad y la fabricabilidad de la PCB, KiCad permite establecer y verificar reglas de diseño, tanto eléctricas como físicas. Estas reglas pueden configurarse globalmente o por clases de señales, y se validan mediante herramientas de verificación integradas.

1. Reglas Eléctricas (ERC)

Aplican al esquemático:

  • Verificar que todos los pines de alimentación estén definidos.
  • Asegurarse de que no haya pines sin conexión cuando deberían estar conectados.
  • Comprobar que las referencias de los componentes sean únicas y coherentes.

2. Reglas de Diseño (DRC)

Aplican al layout de la placa:

  • Ancho mínimo de pistas: depende de la corriente que circulará por ellas.
  • Separación mínima entre pistas y áreas de cobre (clearance): evita cortocircuitos y mejora la confiabilidad.
  • Tamaño mínimo de vías y taladros: asegura que los orificios puedan fabricarse correctamente.
  • Asignación correcta de huellas: los componentes deben encajar físicamente en el PCB.
  • Espacios en máscara de soldadura y serigrafía: deben respetarse para evitar puentes de soldadura o texto ilegible.

3. Clases de Nets

KiCad permite agrupar señales en net classes para aplicar reglas específicas, por ejemplo:

  • Señales de potencia con pistas más anchas.
  • Señales de alta frecuencia con restricciones de distancia.
  • Líneas de señal estándar con reglas más flexibles.

4. Verificación Automática

  • ERC (Electrical Rules Check): analiza errores lógicos en el esquemático.
  • DRC (Design Rules Check): comprueba que el PCB cumpla las restricciones físicas de diseño.
  • Vista 3D: asegura la correcta disposición mecánica y evita colisiones entre componentes.

Consejos para principiantes

  • Comienza con diseños simples antes de abordar proyectos complejos
  • Utiliza siempre la verificación ERC y DRC antes de generar archivos de fabricación
  • Organiza tus componentes en bibliotecas personales para reutilizarlos en futuros proyectos
  • Aprovecha la comunidad de KiCad para resolver dudas y encontrar componentes
  • Experimenta con la vista 3D para detectar problemas de colocación de componentes
  • Configura las reglas de diseño según los requerimientos de tu fabricante de PCB
